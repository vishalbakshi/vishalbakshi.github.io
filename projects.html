San Francisco Transit Riders (SFTR) DataJAM

During this one-day hackathon/datathon hosted by SFTR, Code For America and the American Planning Association, my team (including a Product Manager from Facebook, Data Analyst from Nielsen, Data Scientist from 6Sense, and Executive Director of SFTR) created a Proof Of Concept for a tool that SFTR can use to present visualizations to advocate for their 30 x 30 Rapid Rider Network initiative.

My contributions to this project:

  • Day of Event
    • Facilitate and document initial our brainstorm discussion
    • Set up a local version of the existing open source project: Mapnificent
    • Understand and articulate the existing functionality and app architecture to the team
    • Set up a local version of the open source Mapnificent Generator to convert gtfs tab-delimited files to binary
  • Two Weeks after
    • Research and document existing and additional Mapnificent app functionality to determine our design and development options moving forward
    • Build a local Go server that requests GTFS data and converts it to the structured binary file
  • Four Weeks after
    • Build a local Node server that creates table schema, connects to a Google Cloud SQL instance, and loads GTFS tab-delimited data into those tables
    • Modify Mapnificent's functionality to allow users to select and view individual SFMTA MUNI route isochrones
    • Coordinate, prepare and facilitate a team-wide meeting to finalize the project vision and objectives

Call For Code: Wildfire Challenge

During this one-day hackathon hosted by IBM and 42 Silicon Valley, my team (including a recent UX Design bootcamp graduate and web development bootcamp graduate) created a Proof Of Concept for a tool that United Way could use to manage wildfire victim data. We called this tool the "Wildfire Victim Management System".

My contributions to this project:

  • Day of Event
    • Interview the United Way Development Director to collect information on their existing process to evaluate home damage for cash grant distribution
    • Train a three class (Burned, Not Burned, Negative) image classification model using Watson Studio
    • Build a local Node server that sends an image to the Watson model's API and receives both a class and a score
    • Build a UI which allows a user to submit an image and view the Watson model results
