San Francisco Transit Riders (SFTR) DataJAM

SFTR wanted a visualization tool, for a transit improvement advocacy initiative, to show users areas they can reach with San Francisco public transport in 30 minutes. We forked an existing open source project (Mapnificent) and modified the necessary data structures, data pipeline, and canvas rendering logic to achieve their goal.

Our latest MVP: https://sftrembed.firebaseapp.com/

My contributions:

  • Day One
    • Facilitate and document initial our brainstorm discussion
    • Set up a local version of the existing open source project: Mapnificent to present a Proof Of Concept to the organization
    • Understand and articulate to the team the existing project's functionality and app architecture
    • Set up a local version of the open source Mapnificent Generator to convert gtfs tab-delimited files to binary
  • Week One
    • Research and document existing and additional Mapnificent app functionality to determine our design and development options moving forward
    • Build a local Go server that requests GTFS data and converts it to the structured binary file
  • Week Three
    • Build a local Node server that creates table schema, connects to a Google Cloud SQL instance, and loads GTFS tab-delimited data into those tables
    • Modify Mapnificent's functionality to allow users to select and view individual SFMTA MUNI route isochrones
    • Coordinate, prepare and facilitate a team-wide meeting to finalize the project vision and objectives
  • Week Four
    • Build functionality to render individual route isochrones with different fill colors in order to visually compare the transit reach over local and rapid MUNI routes
    • Deploy the first version of our MVP in preparation for the DataJAM's follow up event on Monday May 13, 2019
  • Week Eleven
    • Update UI to match SFTR and 30x30 color scheme and add logos to each page
    • Generate DOM elements for rapid route selection based on a given array of route IDs
    • Update Rapid Route checkboxes to fall under the same boostrap styling as the rest of the navbar elements
    • Add credits for Mapnificent to the About page along with a brief explanation of the tool and SFTR's 30x30 initiative

Call For Code: Wildfire Challenge

During this one-day hackathon hosted by IBM and 42 Silicon Valley, my team (including a recent UX Design bootcamp graduate and web development bootcamp graduate) created a Proof Of Concept for a tool that United Way could use to manage wildfire victim data. We called this tool the "Wildfire Victim Management System".

My contributions:

  • Day of Event
    • Interview the United Way Development Director to collect information on their existing process to evaluate home damage for cash grant distribution
    • Train a three class (Burned, Not Burned, Negative) image classification model using Watson Studio
    • Build a local Node server that sends an image to the Watson model's API and receives both a class and a score
    • Build a UI which allows a user to submit an image and view the Watson model results
  • Week One
    • Gather requirements from the United Way Development Director and Applications Manager to finalize the first MVP version's user stories and core functionality
    • Establish a development repository
    • Establish the initial structure for the repository's Wiki
  • Week Three
    • Review and provide feedback on the initial Sketch prototypes created by our UX designer
    • Receive further clarifications and features requested by the client on the dashboard and survivor application
  • Week Five
    • Create initial development branch for the Wildfire Survivor Management System (view it on Codesandbox)
