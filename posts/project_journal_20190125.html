Project Journal: January 25, 2019

I had a fantastic today.

I went to the Nodeschool SF meetup which was fantastic. There was an insightful session on Continuous Integration, Continuous Development and Test-Driven-Development that made me rethink how I write and plan code. I seem to have fell into a habit of writing tests after writing code, and I need to reverse that ASAP. It just felt like such a less-stressful, more structured, heavenly way to write code. The presenter (DJ Gill) walked through some simple examples on this repo, and watching how logical it was to write the tests first made me feel quite silly for not doing so all along. It was more methodical, pragmatic, mature of a process than just my constant feeling-anxious-about-not-knowing-if- this-code-works writing style. Gotta write tests first.

In addition to that event's fantasticness, I was able to decide on "non-negotiable" tasks that I needed to get done this weekend in order to have a shot at getting all Four Week Versions "Done" in the fourth week (which is this coming week). These were non-negotiable not in the sense of a "whatever it takes" do-not-sleep-until-finished overnight scramble, but rather a simplify-it-more mindset which would allow me to think more and code less to get the tasks done.

The Weekend Sprint

et130webapp

I want to have my students start using this app soon, so I am starting to prioritize:

  • Hide answers by default
  • Allow user input
  • Notify user if their input is correct or incorrect

ThoughtBoard

I want to have a couple of friends, who might find this app interesting, test it out so I can receive some user feedback soon, so here are this weekend's priorities:

  • Use "tags" instead of "categories" to avoid default/undesirable/confusing/distracting ordering and sorting
  • Replace sorting with a "highlight thoughts by tag" feature
  • Create 2D thought elements to help the user better visualize the process

FeelsTracker

I have been feeling a bit less excited about this one recently, and I want to get that curiosity back, so here are its weekend priorities:

  • Allow user to click and submit a single petal/emotion
  • Finish element which allows user to choose "depth" of the emotion
  • Allow user to submit a "depth" emotion

I'll be providing more detail on all these tomorrow, once I'm done, but I wanted to lay it out there to help me wrap my head around them a little more.
