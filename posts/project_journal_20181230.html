Takeaways from promise-it-wont-hurt

promise-it-wont-hurt was a challenging but fantastic workshopper that walked through different aspects of Promises in JavaScript. Some key takeaways, besides the fundamentals that I won't bother trying to write more effectively than MDN:

  • When writing a new Promise that should be resolved after other promises are resolved, nest your resolve function call inside the then call of the other promises.

An example: 


    var firstPromise = // some promise
    var secondPromise = // some other promise
    
    function onReject() {
        // error handler
    }

    // New user-defined Promise to resolve after other ones
    var thirdPromise = new Promise(function(resolve, reject) {
        let counter = 0;

        firstPromise.then(function(result) {
            counter++;
            if (counter == 2) {
                resolve('newPromise has been resolved!')
            }
        }).catch(onReject);

        secondPromise.then(function(result) {
            counter++;
            if (counter == 2) {
                resolve('newPromise has been resolved!')
            }
        }).catch(onReject);
    });

    thirdPromise.then(console.log)
    // `newPromise has been resolved!` logs on the console

  • You don't know how long a promise will take to resolve
I have to keep reiterating this to myself since I keep forgetting it's true. Coming from a synchronous perspective, it's easy to get lulled into thinking:

firstPromise resolves first because it's first, and secondPromise resolves second because it's second.

The implications of this in the code block above are that the condition if (counter == 2) has to be written in the then call for both promises.

  • JSON.parse() accepts Buffers
This takeaway has less to do with promises, but I learned this nonetheless during one of the workshopper's challenges. A JSON object was received from a GET request and was to be logged in the console. I was initially going to convert the Buffer to a String, but thought I should try it first as is, and it worked.

Takeaways from test-anything

test-anything was a shorter and less challenging workshopper, but still fruitful. The tape package was used for the challenges. Here are my key takeaways:

  • You can set how many tests are going to be done with t.plan(numberOfTests) before executing the tests. If you do not, you must call t.end after the tests have ran. t is the test object that require('tape')([name], [opts], cb)provides in the callback cb.
  • To test if a function called a callback, you can send a test method as a callback

    • 
    var test = require('tape');
    var functionToTest = // function that takes a cb

    test('Testing for callbacks', function(t) {
        t.plan(1);
        functionToTest(function() {
            t.pass('Callback was called')
        });

    });
  • You can test if a function did or did not throw an Error:

    • 
    var test = require('tape');
    var functionToTest = // function that throws an error 
    var noErrorValue = // value that does not cause an error
    var errorCausingValue = // value that causes an error

    test('Checking for exceptions', function(t) {
        t.throws(function() {
            functionToTest(errorCausingValue);
        });

        t.doesNotThrow(function() {
            functionToTest(noErrorValue);
        });

        t.end();

    });
