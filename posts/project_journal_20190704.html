Web APIs

In the SFTR Embed project (forked from Mapnificent) I had implemented a jQuery script which gathered the IDs of user-selected routes in an array and saved it to a global variable, which would later be used by another client-side script to filter the transit data.

Here's the original code:


        // Populate `window.checkRoutes` with an Array of user-selected route IDs
        let routes = [];
        window.checkedRoutes = [];
        $('.route-checkbox').change(function(){
            if ($(this).is(':checked')) {
                routes.push($(this)[0].value);
                window.checkedRoutes = routes;
            } else {
                let routeIdx = routes.indexOf($(this)[0].value);
                routes.splice(routeIdx,1);
                window.checkedRoutes = routes;
            };
            // ...mapnificent function calls to render the filtered isochrones
        })

I wanted to first write tests for this before implementing it into the codebase, but I also wanted to get a move on refactoring it using Web APIs, so I wrote out the following as a first draft:
