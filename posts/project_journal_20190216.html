Project Journal: February 16, 2019

Blast the trumpets for Puppeteer.

I had a very positive, and relatively smooth experience last week testing Puppeteer locally, configuring the travis.yml file, and getting the Travis build to pass.

The following code was all that was needed in ensuring that I checked the QUnit tests, and logged them accordingly, exiting the process with an error code if needed: 


const puppeteer = require('puppeteer');

(async () => {
    const browser = await puppeteer.launch();
    const page = await browser.newPage();

    page.on('console', function(msg){
        for (let i = 0; i < msg.args().length; ++i){
            console.log(`${i}: ${msg.args()[i]}`);
        }
    })

    await page.goto('http://localhost:8080/test');

    // Grab the element based on QUnit-given selector.
    // This element holds the number of failed tests
    const failedSpan = await page.$('span[class=failed]');

    // Log the number of fail tests (if there are any)
    // and exit process with error code
    const failedCount = await page.evaluate(function(failedSpan){
        if (failedSpan.textContent != 0){
            console.log(failedSpan.textContent);
            process.exit(1);
        } else {
            console.log('All QUnit tests have passed');
        }
    }, failedSpan)

    await browser.close();
})();

This worked beautifully, both locally and on Travis.
