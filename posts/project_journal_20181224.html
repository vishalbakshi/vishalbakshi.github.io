Project Journal: December 24, 2018

Today, I started my first attempt at open source contribution. The "first-timers-only" and similarly labelled issues on GitHub was what I was looking for. This repo has an awesome list of such open issues.

The first project I am attempting to contribute to is Public Lab's Image Sequencer. The particular issue which is flagged as first-timers-only is to create a test for one of their modules which converts an image file to QR code. The jsqr package has updated from its 0.2.2 version, and including that update without a test would leave them blind to anything that may break because of it.

I started by forking the repo to my GitHub account, cloning it onto my local computer, and practicing the git fetch upstream command. Initially as I was following the online GitHub documentation (which is great by the way) which showed the git fetch upstream master command, I ran into issues (i.e. that branch was not found) because the default branch on the upstream repo, which I had forked, was the main branch. The solution was, of course, that I had to fetch the main branch from upstream. Ta-da!

I then created a new branch, decodeqr-test, where I will start writing the tests.

Public Lab is using the tape package, which uses the "tap" philosophy, which stands for test anything. I was able to make a basic initial test run: 

                    test('Initial test', function(t) {
                        t.equal(true,true,'Initial test if true === true');
                        t.end();
                    })
which was a feat in itself, since it took me awhile that I had to npm run setup in order to install the dependencies in their project before I could require them!

My next tasks on this project are:

  • Understand what the DecodeQR module does
  • Understand the difference between "unit tests" and "functional tests"
  • Write one test for each unit
  • Write one test for each functionality
